# Exercise 03 — pv (pipe viewer)

**Goal:** see throughput, elapsed time, and a progress bar for any pipeline — without changing the commands on either side of it.

## Setup

```bash
cd demo
sudo apt install pv   # if not already installed
```

## Basic progress bar

```bash
pv data/prompts.jsonl > /dev/null
```

Output:
```
2000 lines  47.2KiB 0:00:00 [1.23MiB/s] [================================>] 100%
```

## Drop pv into any pipeline

```bash
cat data/prompts.jsonl | pv | python3 -c "
import sys
for line in sys.stdin:
    pass  # simulate processing
print('done')
"
```

`pv` is transparent — it passes data through unchanged and prints progress to stderr.

## Monitor line rate for JSONL processing

```bash
pv --line-mode data/prompts.jsonl | python3 -c "
import sys, json
for line in sys.stdin:
    obj = json.loads(line)
" 
```

`--line-mode` counts lines instead of bytes — shows "N lines/s", which is more meaningful for structured data.

## Rate-limit a pipeline

Cap throughput so a fast reader doesn't overwhelm a slow consumer:

```bash
pv --rate-limit 100 data/prompts.jsonl | python3 -c "
import sys, json, time
for line in sys.stdin:
    obj = json.loads(line)
    # simulate slow processing
"
```

`--rate-limit 100` caps at 100 bytes/s. Use `--line-mode --rate-limit 10` for 10 lines/s.

## ETA for a long inference batch

When you know the total line count upfront, `pv` gives an accurate ETA:

```bash
LINES=$(wc -l < data/prompts.jsonl)
pv --line-mode --size "$LINES" data/prompts.jsonl | your-inference-tool
```

## The insight

Every pipeline is a black box until you put `pv` in it. For long-running AI batch jobs — embedding a dataset, running inference over thousands of prompts, writing a large JSONL output — `pv` turns a blinking cursor into a progress bar with an ETA, at zero cost to the surrounding pipeline.
