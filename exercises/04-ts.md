# Exercise 04 — ts (timestamp)

**Goal:** prepend a timestamp to every line of stdin — turning any stream into a timestamped log.

## Setup

```bash
cd demo
sudo apt install moreutils   # provides ts, sponge, vidir
```

## Basic timestamping

```bash
cat logs/agent.log | ts
```

Output:
```
May 25 13:01:02 [INFO] Agent initialising
May 25 13:01:02 [INFO] Loading tool registry
...
```

Each line gets a wall-clock prefix as it passes through.

## Custom format

```bash
cat logs/agent.log | ts '[%H:%M:%S]'
```

```
[13:01:02] [INFO] Agent initialising
[13:01:02] [INFO] Loading tool registry
```

`ts` accepts any `strftime` format string.

## Relative timestamps — time since last line

```bash
cat logs/agent.log | ts -s '%.s'
```

`-s` switches to relative mode. `%.s` gives fractional seconds since the previous line — reveals where time was actually spent in a sequential log.

## Timestamp a long-running process live

```bash
# Pattern: run a slow command and timestamp every line of its output
your-agent-command 2>&1 | ts '[%Y-%m-%dT%H:%M:%S]' | tee outputs/agent-timestamped.log
```

The `2>&1` merges stderr into stdout so both streams get timestamps and land in the same log file.

## Simulate it with the demo log

```bash
# replay the log with artificial delays, timestamp each line
while IFS= read -r line; do
  sleep 0.2
  echo "$line"
done < logs/agent.log | ts '[%H:%M:%S.%.s]' | tee outputs/agent-timed.log

cat outputs/agent-timed.log
```

## The insight

`ts` answers "I wish I knew when each line of this output happened" — after the fact is too late. Pipe through `ts` at the start of any agent run and every log line is self-dated. Combine with `tee` to get timestamped output on screen *and* in a file simultaneously.
